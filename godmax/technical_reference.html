Overview & Design Philosophy

GODMAX is a fully differentiable JAX-based halo model framework. All arrays are JAX arrays unless noted otherwise. The covariance matrices (get_cov) are the only component that uses NumPy/SciPy, because they are not differentiated.

Key design choices:

  • Single-inheritance chain: each class extends the previous; parent objects are passed as *_obj arguments and copied via self.__dict__.update() to avoid recomputation.
  • All shapes frozen at construction time: because methods are @partial(jit, static_argnums=(0,)), array dimensions from halo_params_dict cannot change after instantiation.
  • Grid-indexed vectorization: get_vmapped_func(func, n) builds nested vmap calls over integer index arrays (jnp.arange(nz), etc.), not the physical values directly.
  • jax_enable_x64 is required: halo model integrals are numerically sensitive to float32 rounding.

Full Modeling Pipeline

1. Cosmology & grids — base_class JAX

The constructor calls read_all_input() which ingests all four parameter dicts, then calls get_power_spectra_cosmo().

Cosmological background

A jax_cosmo.Cosmology object is created. Then:

  • plin_kz_mat[nk, nz] — linear matter power spectrum via jax_cosmo or symbolic regression
  • chi_z_array, dchi_dz_array — comoving distances and their z-derivatives
  • Da_z_array — angular diameter distances
  • growth_factor_z_array — linear growth factor D(z)
  • rho_m_bar — mean matter density today (M_sun/Mpc³): $\rho_{m,0} = \Omega_m \cdot \rho_{\rm crit,0}$

Critical and matter densities as functions of redshift:

$$E(z) = \sqrt{\Omega_m(1+z)^3 + \Omega_\Lambda}, \qquad \rho_c(z) = \rho_{\rm crit,0}\,E(z)^2, \qquad \rho_m(z) = \rho_{m,0}(1+z)^3$$

2. Radial profiles — Profiles JAX

Profiles are computed on the full 3D grid [nr, nz, nM]. The initialization calls these methods in order:

  1. setup_hmf() — σ(M,z) and ν(M,z)
  2. get_hmf() — halo mass function dN/dlnM
  3. get_conc_Mz() — concentration–mass relation c(M,z)
  4. setup_main_calc() — gas geometry parameters θej(M,z), θco(M,z)
  5. get_DMO_profiles() — NFW profiles
  6. run_stars_calc() — stellar density (HOD or simple)
  7. run_gas_calc() — gas density profiles (BCMP)
  8. run_clm_calc() — collision-less matter with backreaction
  9. run_dmb_calc() — total dark matter + baryons
  10. run_pressure_calc() — electron pressure + Compton-y (if model_tSZ=True)

Halo mass function

The velocity dispersion σ(M,z) is computed from the linear power spectrum:

$$\sigma^2(M,z) = \frac{1}{2\pi^2}\int_0^\infty k^2\,P_{\rm lin}(k,z)\,\tilde{W}^2(kR)\,\dd k, \qquad \tilde{W}(x) = \frac{3(\sin x - x\cos x)}{x^3}$$
computed via logspace trapezoidal integral, N=64 points

Peak height: $\nu(M,z) = \delta_c / \sigma(M,z)$ with $\delta_c = 1.68647$ (hardcoded).

Two HMF models are available via hmf_model:

Tinker 2008 (T08) — dN/dlnM

Uses a fit to simulations for halo overdensity Δ200c. Parameters α, A, a, b, c are tabulated at specific Δ values and interpolated.

$$\frac{\dd N}{\dd\ln M} = -\frac{\rho_m}{M}\frac{\dd\ln\sigma^{-1}}{\dd\ln M}\,f(\sigma), \qquad f(\sigma) = A\left[\left(\frac{\sigma}{b}\right)^{-a} + 1\right]e^{-c/\sigma^2}$$
Tinker 2010 (T10) — dN/dlnM [default]

Includes explicit redshift evolution. Uses ν = δc/σ formulation:

$$f(\nu) = \alpha\left[1 + (\beta\nu)^{-2\phi}\right]\nu^{2\eta}\exp\!\left(-\tfrac{\gamma}{2}\nu^2\right)$$

Parameters α, β, γ, φ, η depend on Δ and redshift.

NFW profile

$$\rho_{\rm NFW}(r) = \frac{\rho_s}{(r/r_s)(1+r/r_s)^2}, \qquad r_s = r_{200c}/c$$

Optional truncation (nfw_trunc=True, default): smooth cutoff applied at $r_t = \varepsilon_{rt}\,r_{200c}$:

$$\rho_{\rm NFW,trunc}(r) = \rho_{\rm NFW}(r) \cdot \left[1 + (r/r_t)^2\right]^{-2}$$
ε_rt = 4.0 (hardcoded default, overridable via epsilon_rt)

Gas profile (BCMP — Schneider+19 / Pandey+24)

A two-component profile with an inner core and an outer ejected component:

$$\rho_{\rm gas}(r) \propto \frac{1}{\left[1 + (r/r_{\rm co})^2\right]^{\beta/2} \left[1 + (r/r_{\rm ej})^{\gamma_\rho}\right]^{(\delta-\beta)/\gamma_\rho}}$$
γ_rhogas = 2.0, δ_rhogas = 7.0 are hardcoded defaults (overridable)

The ejection radius and core radius evolve with halo mass and redshift:

$$r_{\rm ej}(M,z) = \theta_{\rm ej}(M,z)\cdot r_{200c}, \qquad \theta_{\rm ej} = \theta_{{\rm ej},0}\left(\frac{M}{M_{*,\rm ej}}\right)^{\nu_{\theta M}}\!(1+z)^{\nu_{\theta z}}$$ $$r_{\rm co}(M,z) = \theta_{\rm co}(M,z)\cdot r_{200c}, \qquad \theta_{\rm co} = \theta_{{\rm co},0}\left(\frac{M}{M_{*,\rm co}}\right)^{\nu_{\rm co,M}}\!(1+z)^{\nu_{\rm co,z}}$$

The outer slope parameter β evolves with mass:

$$\beta(M,z) = \frac{3\,(M/M_c)^{\mu_\beta}}{1 + (M/M_c)^{\mu_\beta}}, \qquad M_c = 10^{\log_{10}M_{c,0}}\left(\frac{M}{10^{\log_{10}M_{*,0}}}\right)^{\nu_M}(1+z)^{\nu_z}$$

Backreaction (adiabatic contraction)

When backreaction=True, the dark matter halo is compressed in response to baryons. The compression factor ζ is found by solving:

$$\frac{r_f}{r_i} - 1 - a_\zeta\!\left[\left(\frac{M_i(r_i)}{M_f(r_f)}\right)^{n_\zeta} - 1\right] = 0$$
Solved iteratively; M_i = M_NFW(r_i), M_f = f_clm·M_NFW(r_f/ζ) + M_cga(r_f) + M_gas(r_f)

Pressure profile & Compton-y

Thermal pressure is derived under hydrostatic equilibrium with non-thermal pressure support fraction $f_{nt}$:

$$f_{nt}(r) = \alpha_{nt}\left(\frac{r}{r_{500c}}\right)^{\beta_{nt}}(1+z)^{n_{nt}}, \qquad P_{\rm th} = P_{\rm tot}(1-f_{nt})$$

Electron pressure: $P_e = P_{\rm th}/1.932$ (hardcoded mean molecular weight factor).

3D Compton-y parameter (stored in y3d_mat[nr, nz, nM]):

$$y_{\rm 3d}(r) = \frac{\sigma_T}{m_e c^2}\int P_e\,\dd\ell, \qquad \text{coefficient} \approx 5.03\times10^{-10}\;\text{(Mpc to cm conversion)}$$

HOD galaxy model (Leauthaud+2011 SHMR)

Central occupation:

$$N_{\rm cen}(M_h) = \frac{1}{2}\left[1 - \text{erf}\!\left(\frac{\log_{10}M_{*,\rm thresh} - \log_{10}M_*(M_h)}{\sqrt{2}\,\sigma_{\log M_*}}\right)\right], \quad \sigma_{\log M_*} = 0.25\text{ (default)}$$

Satellite occupation:

$$N_{\rm sat}(M_h) = N_{\rm cen}(M_h)\left(\frac{M_h}{M_{\rm sat}}\right)^\alpha \exp\!\left(-\frac{M_{\rm cut}}{M_h}\right)$$

The threshold mass $M_{*,\rm thresh}$ is solved numerically (binary search) to match the input comoving galaxy number density nbar_gal_comoving_val.

3. 3D power spectra — get_Pkz JAX

FFTLog profile transform

Real-space profiles are Fourier-transformed via mcfitjax.xi2P (FFTLog with lowring=True):

$$\tilde{u}(k) = 4\pi\int_0^\infty r^2\,\frac{\rho(r)}{M_{\rm tot}}\,j_0(kr)\,\dd r$$
Computed for: dmb, NFW, CLM (galaxies), y (tSZ), n_e (electrons)

The FFTLog output is on a non-uniform k-grid; it is then log-log interpolated onto kPk_array.

Galaxy occupation kernels

Two distinct galaxy Fourier kernels are built from the CLM (collision-less matter) profile $\tilde{u}_{\rm clm}(k)$:

$$\tilde{u}_{g,\rm cross}(k) = \frac{N_{\rm cen} + N_{\rm sat}\,\tilde{u}_{\rm clm}(k)}{\bar{n}(z)}, \qquad \tilde{u}_{g,\rm auto}(k) = \frac{2N_{\rm cen}N_{\rm sat}\,\tilde{u}_{\rm clm}(k) + [N_{\rm sat}\,\tilde{u}_{\rm clm}(k)]^2}{\bar{n}(z)^2}$$
u_g_cross used for P_gm cross-terms; u_g_auto used for P_gg 1-halo term (Poisson shot noise included)

Large-scale 2-halo matter bias correction

When do_corr_2h_mm=True, mass conservation is enforced: halos below M_min contribute a white-noise correction to the bias:

$$b_{m,\rm LS}(z) = \int_{M_{\rm min}}^{M_{\rm max}} \frac{M}{\rho_{m,0}}\,\frac{\dd N}{\dd\ln M}\,b(M,z)\,\dd\ln M, \qquad b_{\rm corr} = 1 - b_{m,\rm LS}$$
b_dmb_kz = b_2h(k,z) + (1 - b_LS); applied to both dmb and NFW

1-halo and 2-halo terms

$$P_{\rm 1H}(k,z) = \int_{M_{\rm min}}^{M_{\rm max}} \tilde{u}_A(k,M,z)\,\tilde{u}_B(k,M,z)\,\frac{\dd N}{\dd\ln M}\,\dd\ln M$$ $$P_{\rm 2H}(k,z) = b_A(k,z)\,b_B(k,z)\,P_{\rm lin}(k,z)$$

The 2-halo bias for each field X is:

$$b_X(k,z) = \int \tilde{u}_X(k,M,z)\,b(M,z)\,\frac{\dd N}{\dd\ln M}\,\dd\ln M$$

The Tinker 2010 halo bias (used for all fields) is:

$$b(\nu) = 1 - \frac{A\nu^a}{\nu^a+\delta_c^a} + B\nu^b + C\nu^c$$
Parameters A, a, B, b, C, c depend on log₁₀Δ (see Tinker et al. 2010)

4. Angular power spectra — get_Cl JAX

Limber projection

$$C_\ell^{AB} = \int_0^{z_{\rm max}} \dd z\,\frac{\dd\chi}{\dd z}\frac{1}{\chi^2(z)}\,W_A(z)\,W_B(z)\,P_{AB}\!\left(k = \frac{\ell+\tfrac{1}{2}}{\chi(z)},\, z\right)$$
Integrated over z_array_for_Cls using trapezoidal rule

The 3D power spectra are evaluated via 2D log-log interpolators (interpax.Interpolator2D) in (log ℓ, z) space.

Window functions

Beam suppression

A Gaussian beam is applied to tSZ-containing power spectra:

$$B_\ell = \exp\!\left(-\frac{\ell(\ell+1)\sigma_{\rm beam}^2}{2}\right), \qquad \sigma_{\rm beam} = \frac{\theta_{\rm FWHM}[\rm arcmin]}{60\,\sqrt{8\ln 2}\cdot(180/\pi)}$$
  • ky cross-spectrum: beam power 1 — $P_{ky}(\ell,z) \to P_{ky}\cdot B_\ell$ (applied to Pkym_lz_mat)
  • gy cross-spectrum: beam power 1 — $P_{gy}(\ell,z) \to P_{gy}\cdot B_\ell$ (applied to Pkgy_lz_mat)
  • yy auto-spectrum: beam power 2 — $P_{yy}(\ell,z) \to P_{yy}\cdot B_\ell^2$ (inside get_Pkyy_lz)

tSZ auto-spectrum noise handling

The yy noise is not computed from first principles. Three cases:

  1. yy_total_ell_fname provided → load Cℓ,yytot from file; noise = total − signal
  2. yy_noise_ell_fname provided → load noise; total = signal + noise
  3. Neither → noise = 0; total = theory only (prints warning)

Cached power spectra

cached_power_spectra[4, 4, nell, nz_for_Cls] stores the Limber-projected Pℓz for fast access in covariance calculations. It is populated as:

[0,0] = P_κκ (baryonified matter)    [1,1] = P_κκ,NFW (halofit)
[0,3] = P_κy × B_ℓ                   [3,3] = P_yy × B_ℓ²
[2,0] = P_gκ                          [2,2] = P_gg
[2,3] = P_gy × B_ℓ                    [2,1] = P_g,κ_NFW

5. Covariance matrices — get_cov NumPy / SciPy

Not JAX-differentiated. The covariance computation uses numpy and scipy.integrate.simpson. This is intentional: covariances are treated as fixed (data compression) and are not differentiated.

Gaussian (Knox) covariance

$$\mathrm{Cov}_G^{(AB)(CD)}(\ell) = \frac{\delta_{\ell\ell'}}{f_{\rm sky}(2\ell+1)\Delta\ell}\left[\tilde{C}_\ell^{AC}\,\tilde{C}_\ell^{BD} + \tilde{C}_\ell^{AD}\,\tilde{C}_\ell^{BC}\right]$$
where $\tilde{C}_\ell = C_\ell^{\rm signal} + N_\ell^{\rm noise}$; Δℓ = band width of each ℓ bin

Non-Gaussian (trispectrum) covariance

$$\mathrm{Cov}_{NG}^{(AB)(CD)}(\ell_1,\ell_2) = \frac{1}{4\pi\,f_{\rm sky}}\,T^{ABCD}(\ell_1,\ell_2)$$ $$T^{ABCD}(\ell_1,\ell_2) = \int\!\dd z\,\frac{\dd\chi}{\dd z}\frac{1}{\chi^6(z)}\int\!\dd\ln M\,\frac{\dd N}{\dd\ln M}\,u_A^{\ell_1}\,u_B^{\ell_1}\,u_C^{\ell_2}\,u_D^{\ell_2}$$
u_X^ℓ = profile kernel at k_ℓ(z) weighted by window; evaluated at (ℓ+0.5)/χ(z)

Profile kernels for trispectrum (pre-computed at init):

  • uy_l_for_cov: $u_y(\ell,z,M) = \tilde{u}_y(k_\ell,z,M)\cdot B_\ell \cdot W_y(z)/\chi^2(z)$
  • ukappa_l_for_cov[nbins]: $u_\kappa(\ell,z,M,b) = \tilde{u}_{\rm dmb}(k_\ell,z,M)\cdot M_{\rm tot}/\rho_{m,0}\cdot W_\kappa(z,b)/\chi^2(z)$
  • ug_l_for_cov[nbins_lens]: $u_g(\ell,z,M,b) = \tilde{u}_{g,\rm cross}(k_\ell,z,M)\cdot W_g(z,b)/\chi^2(z)$

Total covariance

$$\mathrm{Cov}_{\rm tot} = \mathrm{Cov}_G + \mathrm{Cov}_{NG}$$

The result is symmetrized and negative eigenvalues are clipped to zero.

Real-space covariance (two-Bessel transform)

When analysis_coords='real', the Fourier-space covariance is converted to angular correlation function space via:

$$\mathrm{Cov}_\xi(\theta_1,\theta_2) = \int\frac{\dd\ell\,\ell}{2\pi}\,\frac{\dd\ell'\,\ell'}{2\pi}\,J_\nu(\ell\theta_1)\,J_\nu(\ell'\theta_2)\,\mathrm{Cov}_G(\ell,\ell')$$

Implemented via helpers/twobessel.py using FFT-based Hankel transforms with Bessel function orders (ν₁,ν₂): (0,0) for ξ₊, (4,4) for ξ₋, (2,2) for γ_ty.

Sky fraction dictionary

Each probe pair has its own fsky, set in analysis_dict:

fsky_yy, fsky_ky, fsky_kk, fsky_yg, fsky_kg, fsky_gg

6. Real-space correlation functions — get_xi JAX

$$\xi_+(\theta) = \int_0^\infty\frac{\dd\ell\,\ell}{2\pi}\,C_\ell^{\kappa\kappa}\,J_0(\ell\theta), \quad \xi_-(\theta) = \int_0^\infty\frac{\dd\ell\,\ell}{2\pi}\,C_\ell^{\kappa\kappa}\,J_4(\ell\theta), \quad \gamma_t(\theta) = \int_0^\infty\frac{\dd\ell\,\ell}{2\pi}\,C_\ell^{\kappa y}\,J_2(\ell\theta)$$

Computed via mcfitjax.transforms.Hankel with lowring=True, q=1.0. Output is in arcminutes.

Baryonic Suppression: Where It Is and Is Not Applied

The suppression factor is defined as:

$$S(k,z) = \frac{P_{\rm halofit}(k,z)}{P_{\rm NFW,tot}(k,z)} = \frac{P_{\rm halofit}}{P_{\rm 1H,NFW} + P_{\rm 2H,NFW}}$$
SpectrumSuppression applied?Rationale / Note
Pmm (matter auto)YesPdmb × S(k,z). If model_matter='halofit', S is 1 (uses pure halofit)
Pgg (galaxy auto)Yes(P1H+P2H)×S; baryons affect galaxy distribution
Pgm (galaxy × matter)Yes(P1H+P2H)×S
Pge (galaxy × electrons)Yes(P1H+P2H)×S
Pym (tSZ × matter)No (default)Only applied if tSZ_transition_model='response'; otherwise poweradd formula used
Pyy (tSZ auto)NoPure 1H+2H; no suppression factor ever applied
Pgy (galaxy × tSZ)No (default)Same as Pym — poweradd only; response model applies suppression
Design decision — tSZ spectra and suppression: The poweradd model (default) combines 1H+2H via $P = (P_{1H}^\alpha + P_{2H}^\alpha)^{1/\alpha}$ and applies no suppression factor for tSZ-containing spectra. The response model additionally multiplies by S(k,z). The physical motivation is that the tSZ signal is sensitive to the baryon distribution directly via the pressure profile, and the suppression factor was calibrated for matter, not y. Using poweradd avoids double-counting.

Noise Term Definitions

Shape noise (weak lensing κ)

$$N_\ell^{\kappa\kappa}(b,b') = \delta_{bb'}\,\frac{\sigma_\varepsilon^2(b)}{n_{\rm eff}(b)\,\Omega_{\rm pix}}, \qquad \Omega_{\rm pix} = \frac{\pi}{180\times60}\;[\rm sr/arcmin^2]$$
σ_ε per source bin from sigma_epsilon_SN_bins; n_eff per bin from neff_arcmin2_SN_bins

Galaxy shot noise (gg)

$$N_\ell^{gg}(b,b') = \delta_{bb'}\,\frac{1}{\bar{n}_g(b)}$$
n̄_g = comoving number density; solved from nbar_gal_comoving_val by matching HOD

tSZ noise (yy)

The yy noise is loaded from an external file (yy_noise_ell_fname) or derived as the difference between a total map spectrum and the theory signal. It is not computed analytically from instrument noise.

Cross-spectrum noise

Cross-spectra (ky, gy, gk) have zero noise by default (assuming uncorrelated noise between different instruments/surveys).

Known Approximations & Caveats

Limber approximation All C(ℓ) are computed under the Limber approximation $k \approx (\ell+\tfrac{1}{2})/\chi(z)$. This is accurate to ~1% for ℓ > 10 but breaks down at very low ℓ for wide-kernel probes.
1-halo / 2-halo transition (poweradd) The poweradd model $(P_{1H}^\alpha + P_{2H}^\alpha)^{1/\alpha}$ is a phenomenological prescription. At $\alpha=1$ it is the standard linear sum. The transition scale where 1-halo and 2-halo terms are equal is not treated with a physical model (unlike HMcode, which has an explicit transition term). For tSZ-containing spectra, the default is poweradd without baryonic suppression.
Baryonic suppression applied to tSZ only via 'response' model In the default poweradd mode, P_ym and P_yy do not receive the suppression factor S(k,z). This is because the gas profile itself captures the baryon redistribution. Applying S on top would double-count baryonic effects. If debugging tSZ predictions at large k, check which transition model is active.
P_yy receives beam² while P_ky receives beam¹ The yy auto-power bakes in $B_\ell^2$ inside get_Pkyy_lz. The ky cross-spectrum applies $B_\ell^1$ to Pkym_lz_mat (one factor for the y field). This asymmetry is correct physically but easy to miss when comparing spectra.
tSZ noise is not computed from first principles The noise on C_yy must be supplied externally (via yy_total_ell_fname or yy_noise_ell_fname). Without these files, the code proceeds with zero yy noise and prints a warning. This affects the Gaussian covariance diagonal for all probe pairs involving y.
Electron pressure to thermal pressure ratio is hardcoded The factor $P_e = P_{\rm th}/1.932$ assumes fully ionized gas with a fixed helium mass fraction. This is not a free parameter and cannot be changed via the YAML files.
Galaxy number density threshold is solved per-redshift at init time The HOD threshold mass M*,thresh(z) is computed during Profiles.__init__ via binary search. If the target nbar_gal_comoving_val cannot be matched at some redshift (e.g., because M_min is too high), the code may return edge values silently.
Large-scale 2-halo matter bias correction When do_corr_2h_mm=True, the code enforces $\int (M/\rho_m)\,(dN/dlnM)\,b\,dlnM = 1$ by adding a k-independent correction. This is applied to the dmb and NFW (and electron) fields, but not to the tSZ field. The tSZ bias b_y is not corrected.
Backreaction iterative solver resolution The backreaction equation is solved on the same (nr, nz, nM) grid as the profiles. The compression factor ζ(r) is found by linear interpolation of the residual function. At very high or very low M/z, the iteration may not converge smoothly — check rho_clm_mat for NaN or negative values.
Non-Gaussian covariance uses NumPy/SciPy throughout The trispectrum calculation uses scipy.integrate.simpson and NumPy arrays. JAX arrays from the upstream calculations are converted at the boundary. This means the covariance is not differentiable.
Probe index symmetry in cached_power_spectra The 4×4 array is symmetric: [i,j] is set equal to [j,i] immediately after computation. However, the y-beam factor is already baked into the [0,3] and [3,3] entries, so they should not be accessed without awareness that B_ℓ is included.
Photo-z bias shifts n(z) but does not renormalize in k-space get_photoz_biased_nz shifts the source n(z) by Δz and renormalizes the integral. It does not apply a multiplicative shear bias (mult_shear_bias_array is read but not currently applied in the main pipeline — verify in get_Cl if using).

Diving Deeper into Angular Power Spectra

Each C(ℓ) is obtained via the same Limber machinery but the 3D power spectrum $P_{AB}(k,z)$ that enters, the window functions $W_A(z)$, the beam, and — critically — whether the baryonic suppression $S(k,z)$ has been applied to $P_{AB}$, differ per probe. This section documents all of these choices in one place and then derives the self-consistent prescription for $S$ across all six probe cross-combinations.

All spectra share the same Limber kernel:

$$C_\ell^{AB} = \int_0^{z_{\rm max}} \frac{\dd z}{\chi^2(z)}\frac{\dd\chi}{\dd z}\; W_A(z)\,W_B(z)\;P_{AB}\!\!\left(\frac{\ell+\tfrac{1}{2}}{\chi(z)},\,z\right)$$

$\langle\kappa\kappa\rangle$ — CMB / weak-lensing auto JAX

3D power spectrum

Pmm_tot_mat[nk, nz] = baryonified matter power $P_{\rm dmb}(k,z) \times S(k,z)$.
If model_matter='halofit' the full $P_{\rm halofit}$ is used directly (i.e. $S \equiv 1$).

Window function

$$W_\kappa(z, b) = \frac{3H_0^2\Omega_m}{2c^2}(1+z)\,\chi(z)\int_z^{z_{\rm max}}\!\frac{\chi(z')-\chi(z)}{\chi(z')}\frac{\dd n_s^{(b)}}{\dd z'}\dd z' \;-\; A_{\rm IA}\,C_1\,\rho_{m,\rm bar}\,D^{-1}(z)\!\left(\frac{1+z}{1+z_0}\right)^{\!\eta_{\rm IA}}\!\frac{\dd n_s^{(b)}}{\dd z}$$
Lensing kernel (first term) + NLA intrinsic-alignment contribution (second term). Photo-z bias shifts n(z) by Δz before evaluation.

Beam

None. kk does not involve the tSZ field.

Noise

Shape noise — diagonal in source-bin pairs: $N_\ell^{\kappa\kappa}(b,b') = \delta_{bb'}\,\sigma_\varepsilon^2(b)\,/\,n_{\rm eff}(b)$. For GODMAX's Gaussian covariance the kk × kk block additionally gets the B-mode noise correction $N_\ell^{BB}$ (since shape noise contributes equally to E and B modes).

Where it appears in the code

# In get_Cl.__init__:
cached_power_spectra[0, 0, :, :]  = P_kk_lz_mat      # baryonified κ
cached_power_spectra[1, 1, :, :]  = P_kk_nfw_lz_mat   # NFW/halofit κ (if calc_nfw_only=True)

# C(ℓ) assembled via:
Cl_kappa_kappa_tot_mat  = get_Cl_tot(bin1, bin2, 0, 0)

Suppression status

Full $S(k,z)$ applied at the 3D power spectrum level via Pmm_tot_mat = Pmm_dmb × S. The lensing convergence integrates over all matter — dark matter and baryons — so the baryonified spectrum is the physically correct input.

$\langle g\kappa\rangle$ — galaxy × lensing JAX

3D power spectrum

Pgm_tot_mat[nk, nz] = $(P_{\rm 1H,gm} + P_{\rm 2H,gm}) \times S(k,z)$. The galaxy Fourier profile used here is $\tilde{u}_{g,\rm cross}$ (central + satellite × NFW CLM kernel), and the matter profile is $\tilde{u}_{\rm dmb}$.

Window functions

$$W_g(z, b_\ell) = \frac{n_{\rm lens}^{(b_\ell)}(z)}{\int n_{\rm lens}^{(b_\ell)}\,\dd z}, \qquad W_\kappa(z, b_s) = \text{(lensing kernel for source bin }b_s\text{)}$$

Beam

None.

Noise

Zero by default — galaxies and CMB lensing are assumed to have uncorrelated noise.

Where it appears in the code

cached_power_spectra[2, 0, :, :]  = P_gk_lz_mat

Cl_g_kappa_tot_mat  = get_Cl_tot(bin_lens, bin_source, 2, 0)

Suppression status

Full $S(k,z)$ applied. Both the galaxy (NFW-positioned halos) and the matter (lensing) fields need the baryonic correction — both are NFW-based in the halo model. This is the self-consistent result derived in the suppression section below.

$\langle yy\rangle$ — tSZ auto JAX

The full computational chain for $C_\ell^{yy}$ is longer than any other probe because the y field is built from scratch within GODMAX rather than being proxied by an NFW profile. Each step is documented below in order.

Step 1 — BCMP gas density profile → electron pressure

In get_radial_profiles.run_pressure_calc() (called if model_tSZ=True):

  1. The BCMP gas density profile $\rho_{\rm gas}(r;M,z)$ (with ejection and core radii $r_{\rm ej}$, $r_{\rm co}$, slope $\beta$ all mass- and z-dependent) is already stored in rho_gas_mat[nr, nz, nM] from the earlier run_gas_calc() call.
  2. Thermal pressure is derived under hydrostatic equilibrium with non-thermal support:
    $$P_{\rm th}(r;M,z) = P_{\rm tot}(r;M,z)\cdot\bigl[1 - f_{nt}(r;z)\bigr], \quad f_{nt} = \alpha_{nt}\!\left(\frac{r}{r_{500c}}\right)^{\!\beta_{nt}}(1+z)^{n_{nt}}$$
  3. Electron pressure (assuming fully ionized, fixed He fraction):
    $$P_e(r;M,z) = \frac{P_{\rm th}(r;M,z)}{1.932}$$
    Stored as Pe_mat_physical[nr, nz, nM] in units of keV/cm³
  4. 3D Compton-y profile:
    $$y_{\rm 3d}(r;M,z) = \frac{\sigma_T}{m_e c^2}\,P_e(r;M,z)\times \text{Mpc}\to\text{cm}$$
    Coefficient $\approx 5.03\times10^{-10}\,\text{(cm}^3/\text{keV)}\cdot h^{-1}$. Stored as y3d_mat[nr, nz, nM].

Step 2 — FFTLog: real-space y profile → Fourier profile $\tilde{u}_y(k,M,z)$

In get_Pkzs.__init__, immediately after the CLM and electron FFTLogs:

# get_Pkzs.py
k_mcfit, uk_y = xi2P_obj(self.y3d_mat, axis=0, extrap=False)
self.uk_y_tointp = jnp.array(uk_y)     # shape [nk_fftlog, nz, nM]

The FFTLog output uk_y_tointp is on a non-uniform k grid from xi2P. It is then log-log interpolated onto kPk_array and stored as probe index 3 inside get_uk_interp:

# Inside get_uk_interp(jk, jz, jM, probe):
(probe == 3,  self.uk_y_tointp[:, jz, jM])   # tSZ Fourier profile

Step 3 — tSZ bias $b_y(k,z)$ and 2-halo term

In get_Pkzs.__init__, the effective large-scale tSZ bias is computed as the mass-weighted integral of $\tilde{u}_y$:

$$b_y(k,z) = \int_{M_{\rm min}}^{M_{\rm max}} \tilde{u}_y(k,M,z)\,b(M,z)\,\frac{\dd N}{\dd\ln M}\,\dd\ln M$$
by_kz_mat[nk, nz] — computed by vmapping get_b_X(jk, jz, probe=3)

The 2-halo yy power spectrum follows immediately (still in get_Pkzs):

$$P_{yy}^{\rm 2H}(k,z) = b_y(k,z)^2\,P_{\rm lin}(k,z)$$
Pyy_2h_kz_mat = by_kz_mat * by_kz_mat * plin_kz_mat (assembled in get_Cls.__init__)

Step 4 — 1-halo term and total $P_{yy}(k,z)$

Assembled in get_Cls.__init__ (note: unlike $P_{gg}$, $P_{gm}$, etc. which are fully assembled in get_Pkzs, $P_{yy}$ is assembled one class later):

$$P_{yy}^{\rm 1H}(k,z) = \int \tilde{u}_y(k,M,z)^2\,\frac{\dd N}{\dd\ln M}\,\dd\ln M$$
Pyy_1h_kz_mat via get_vmapped_func_warg(get_P_1h, 2, 4)(arange(nk), arange(nz), 3, 3)
$$P_{yy}^{\rm tot}(k,z) = P_{yy}^{\rm 1H}(k,z) + P_{yy}^{\rm 2H}(k,z)$$
Pyy_tot_kz_mat[nk, nz] — no $S(k,z)$ ever applied

Step 5 — Limber projection with beam² baked in

The method get_Pkyy_lz(jl, jz) evaluates $P_{yy}$ at the Limber wavenumber and applies the beam immediately:

$$\tilde{P}_{yy}(\ell, z) = B_\ell^2 \times P_{yy}^{\rm tot}\!\!\left(\frac{\ell+\tfrac{1}{2}}{\chi(z)},\,z\right), \quad B_\ell = \exp\!\!\left(-\frac{\ell(\ell+1)\sigma_{\rm beam}^2}{2}\right)$$
Pkyy_lz_mat[nell, nz]. $B_\ell^2$ because both y fields carry one beam factor.

This is the key difference from all other probes: the beam enters inside get_Pkyy_lz rather than being applied afterward. For ky and gy, $B_\ell^1$ is applied to the Limber-projected matrix after the fact.

Step 6 — 2D log-log interpolator and cache

Step 7 — $C_\ell^{yy}$ signal via Limber integral

$$C_\ell^{yy} = \int_0^{z_{\rm max}} \frac{\dd z}{\chi^2(z)}\frac{\dd\chi}{\dd z}\;W_y(z)^2\;\tilde{P}_{yy}(\ell,z), \qquad W_y(z) = \frac{1}{1+z}$$

Baryonic suppression consistency across probes — derivation

$S$ is defined at the level of the matter auto-spectrum. The question is: what correction does this imply for cross-spectra that mix an NFW-based field (matter, galaxies) with the BCMP gas field (tSZ)? The answer follows directly from the bilinear structure of the halo model.

Step 1 — What $S$ is and what it corrects

$$S(k,z) \equiv \frac{P_{\rm halofit}(k,z)}{P_{\rm NFW,tot}(k,z)} < 1 \quad \text{for } k \gtrsim 0.1\,h\,\text{Mpc}^{-1}$$

The halo model with NFW profiles over-predicts the matter clustering on small scales because it ignores that baryonic feedback (AGN, stellar winds) has ejected gas from halos, redistributing mass to larger radii and reducing the matter power. $S$ quantifies this suppression by comparing to halofit (calibrated on hydro simulations). Applying $S$ to $P_{mm}$ brings the NFW halo model into agreement with the true baryonified matter power spectrum.

Step 2 — $S$ is defined for an auto-spectrum, so it involves squaring a profile correction

In the halo model, all power spectra are bilinear in Fourier-space profiles — linear in $\tilde{u}_A$ and linear in $\tilde{u}_B$ separately:

$$P_{AB}(k,z) = \underbrace{\int n(M)\,\tilde{u}_A(k,M,z)\,\tilde{u}_B(k,M,z)\,\dd\ln M}_{\text{1-halo}} + \underbrace{b_A(k,z)\,b_B(k,z)\,P_{\rm lin}(k,z)}_{\text{2-halo}}$$

The auto-spectrum $P_{mm}$ involves the same profile squared: $\tilde{u}_m^2$ in the 1-halo, and $b_m^2$ in the 2-halo. So when we say $P_{mm}^{\rm true} = S \cdot P_{mm}^{\rm halo}$, we are saying that the effective matter profile is rescaled by $\sqrt{S}$:

$$\tilde{u}_m^{\rm eff} = \sqrt{S(k,z)}\cdot\tilde{u}_m^{\rm NFW}$$
Squaring recovers the auto-spectrum: $[\sqrt{S}\,\tilde{u}_m]^2 = S\,[\tilde{u}_m]^2$, hence $P_{mm}^{\rm eff} = S\cdot P_{mm}^{\rm halo}$ ✓

The key point: $S$ is the correction at the power-spectrum level, but $\sqrt{S}$ is the correction at the field/profile level. The factor of 2 (in log) comes from squaring.

Why $S$ (full) on $P_{gy}$ would be wrong

Applying the full $S$ to $P_{gy}$ would be equivalent to assigning $f_y = \sqrt{S}$ to the y field — i.e. treating the y profile as if it also needs an NFW→baryonified correction. But the y profile is already the baryonified result (BCMP gas). Suppressing it by $\sqrt{S}$ would double-count the baryon redistribution on the y side, making the tSZ signal artificially lower than the gas model predicts.

PSD — why $\sqrt{S}$ is the only consistent prescription

Construct the 3×3 power-spectrum matrix at fixed $(k,z)$:

$$M^{\rm halo} = \begin{pmatrix} P_{kk} & P_{gk} & P_{ky} \\ P_{gk} & P_{gg} & P_{gy} \\ P_{ky} & P_{gy} & P_{yy} \end{pmatrix}^{\rm halo}$$

The halo model guarantees $M^{\rm halo}$ is positive semi-definite — it is a sum of rank-1 outer products $\mathbf{u}\mathbf{u}^T$ where $\mathbf{u} = (\tilde{u}_\kappa, \tilde{u}_g, \tilde{u}_y)$. Applying $f_\kappa = f_g = \sqrt{S}$, $f_y = 1$ to each field is a congruence transformation:

$$M^{\rm corrected} = D\,M^{\rm halo}\,D, \qquad D = \mathrm{diag}\!\left(\sqrt{S},\;\sqrt{S},\;1\right)$$
A congruence $DMD$ with any positive-definite $D$ preserves positive semi-definiteness.

Any other diagonal correction that does not match the $f_A \times f_B$ bilinear structure — such as applying full $S$ to $P_{gg}$ but nothing to $P_{gy}$ and $P_{yy}$ — is not a congruence transformation and can break PSD. Concretely, with the current GODMAX defaults, the Cauchy–Schwarz condition $P_{gy}^2 \leq P_{gg} \cdot P_{yy}$ becomes:

$$\underbrace{\bigl(P_{gy}^{\rm halo}\bigr)^2}_{\text{unchanged}} \leq \underbrace{S \cdot P_{gg}^{\rm halo}}_{\displaystyle P_{gg}} \cdot \underbrace{P_{yy}^{\rm halo}}_{\displaystyle P_{yy}}$$

The halo model only guarantees $(P_{gy}^{\rm halo})^2 \leq P_{gg}^{\rm halo} \cdot P_{yy}^{\rm halo}$, which is weaker by $1/S > 1$. Wherever $S \ll 1$ (small scales, high-mass halos), this fails and the covariance matrix acquires genuine negative eigenvalues — not numerical noise but physical inconsistency from the asymmetric suppression.

The fix

Apply $\sqrt{S(k,z)}$ to Pgy_tot_mat and Pym_tot_mat in get_Pkzs.py:

$$P_{gy}^{\rm corrected} = P_{gy}^{\rm poweradd} \times \sqrt{S(k,z)}, \qquad P_{ym}^{\rm corrected} = P_{ym}^{\rm poweradd} \times \sqrt{S(k,z)}$$

This is the only prescription that: (a) correctly corrects the NFW galaxy/matter side, (b) does not touch the BCMP gas side, and (c) guarantees a PSD covariance matrix at the 3D power-spectrum level before Limber projection.

Note on the response model: tSZ_transition_model='response' applies the full $S$ to $P_{gy}$ and $P_{ym}$. In the congruence framework this corresponds to $D = \mathrm{diag}(\sqrt{S}, \sqrt{S}, \sqrt{S})$, i.e. also suppressing the y field by $\sqrt{S}$. That is equivalent to treating the BCMP profile as if it also needs an NFW correction — double-counting the gas redistribution. The correct prescription is $\sqrt{S}$ (one field-level factor) on the mixed-field cross-spectra only.
