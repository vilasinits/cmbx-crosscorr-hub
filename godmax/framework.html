Introduction

GODMAX is a JAX-based halo model framework for jointly modelling baryonic thermodynamics and the cosmic large-scale structure. It connects the 3D density and pressure profiles of gas inside haloes to observable cross-power spectra, enabling simultaneous inference of cosmological and astrophysical parameters from multi-probe data.

Core scientific problem: Baryonic feedback from AGN and supernovae redistributes gas on megaparsec scales, suppressing the total matter power spectrum by 10–20 % at weak-lensing scales. This is the dominant systematic uncertainty in current cosmological weak-lensing analyses. GODMAX models this self-consistently by solving hydrostatic equilibrium rather than applying a post-hoc correction.

What does the name mean?

Gas thermO Dynamics and Matter distribution using jAX

The name encapsulates the two coupled physics themes (gas thermodynamics + dark matter/baryon distribution) and the computational backbone (JAX for auto-differentiation and JIT compilation).

The framework targets joint analyses of:

  • Weak lensing convergence auto-correlations κκ
  • Convergence × thermal SZ cross-correlation κy
  • Galaxy clustering auto-correlation gg (code only, not paper)
  • Galaxy × convergence gk, Galaxy × tSZ gy (code only)
  • Galaxy-galaxy lensing excess surface density ge (code only)
Paper vs Code gap: The arXiv paper (2401.18072) only analyses $\kappa\kappa$ and $\kappa y$ in real space. The refactor-map branch of the code implements a much wider probe set (gg, gk, gy, ge), which are used in the Fisher/MCMC run scripts but not formally described in the paper.

The target survey combination in the paper is Vera Rubin LSST (weak lensing source catalogue, 5 tomographic bins) × Simons Observatory (tSZ y-maps, 1.0 arcmin beam).

Halo Model

GODMAX uses the standard halo-model decomposition: the total power spectrum is the sum of a one-halo (intra-halo) and a two-halo (inter-halo) term.

3D Power Spectrum

Eq. 29 $$P^{AB}_\mathrm{tot}(k,z) = P^{AB}_\mathrm{1h}(k,z) + P^{AB}_\mathrm{2h}(k,z)$$
Eq. 22 — 1-halo $$P^{AB}_\mathrm{1h}(k,z) = \int_{M_\mathrm{min}}^{M_\mathrm{max}} dM\, \frac{dn}{dM} \int_{c_\mathrm{min}}^{c_\mathrm{max}} dc\; p(c|M)\; u^A(k,z,M,c)\; u^B(k,z,M,c)$$
Eq. 25 — 2-halo $$P^{AB}_\mathrm{2h}(k,z) = b^A(k,z)\; b^B(k,z)\; P_\mathrm{lin}(k,z)$$
where integration limits are $M \in [10^{11}, 10^{16}]\,M_\odot/h$ and concentrations $c \in [2,8]$.

Effective bias (2-halo weight)

Eq. 27–28 $$b^A_\mathrm{eff}(k,z) = \int dM\, \frac{dn}{dM} \int dc\; p(c|M)\; b^\mathrm{lin}(M)\; u^A(k,z,M,c) + A(M_\mathrm{min})$$ $$A(M_\mathrm{min}) = 1 - \lim_{k\to 0}\int dM\, \frac{dn}{dM} \int dc\; p(c|M)\; b^\mathrm{lin}(M)\; u^A$$

The mass conservation term $A(M_\mathrm{min})$ ensures the 2-halo term integrates correctly even when low-mass haloes are excluded.

Gas & Matter Profiles

Total baryonic matter density

Eq. 2 $$\rho_\mathrm{dmb}(r) = \rho_\mathrm{cga}(r) + \rho_\mathrm{gas}(r) + \rho_\mathrm{clm}(r)$$

Central galaxy ($\rho_\mathrm{cga}$) + free gas ($\rho_\mathrm{gas}$) + "cold" (stellar/IGM) component ($\rho_\mathrm{clm}$).

Gas density profile (flexible GNFW)

Eq. 7 $$\rho_\mathrm{gas}(r) = \frac{\rho_{\mathrm{gas},0}} {\left[1+\left(\dfrac{r}{\theta_\mathrm{co}\,r_{200c}}\right)\right]^{\!\beta} \left[1+\left(\dfrac{r}{\theta_\mathrm{ej}\,r_{200c}}\right)^{\!\gamma}\right]^{(\delta-\beta)/\gamma}}$$

Mass-dependent slope $\beta$

Eq. 8–9 $$\beta(M,z) = \frac{3\,(M_{200c}/M_c)^{\mu_\beta}}{1+(M_{200c}/M_c)^{\mu_\beta}}, \qquad M_c = M_{c,0}(1+z)^{\nu_{M_c}}$$

Radius evolution with mass/redshift/concentration

Eq. 10 $$\theta_{\mathrm{ej/co}} = \theta_{\mathrm{ej/co},0} \left(\frac{M_{200c}}{M_\star}\right)^{\!\mu_{\mathrm{ej/co}}} (1+z)^{\nu_{\mathrm{ej/co}}}\; c_{200c}^{-\eta_{\mathrm{ej/co}}}$$

Hydrostatic equilibrium → electron pressure

Eqs. 14–18 $$\frac{1}{\rho_\mathrm{gas}}\frac{dP_\mathrm{tot}}{dr} = -G\frac{M_\mathrm{dmb}(
Key innovation: GODMAX solves the exact hydrostatic equilibrium ODE rather than assuming a parametric GNFW pressure profile. This couples $P_e$ self-consistently to $\rho_\mathrm{dmb}$.

Angular Power Spectra

The Limber approximation ($k = (\ell+\tfrac{1}{2})/\chi$) is used throughout to project 3D power spectra to angular $C_\ell$.

General 1-halo and 2-halo angular terms

Eqs. 30–31 $$C^{ij}_{AB;\,\mathrm{1h}}(\ell) = \int dz\;\frac{dV}{dz\,d\Omega} \int dM\;\frac{dn}{dM}\int dc\;p(c|M)\;\bar{u}^{\,i}_A(\ell,M,z)\;\bar{u}^{\,j}_B(\ell,M,z)$$ $$C^{ij}_{AB;\,\mathrm{2h}}(\ell) = \int dz\;\frac{dV}{dz\,d\Omega}\; b^i_A(\ell,z)\;b^j_B(\ell,z)\;P_\mathrm{lin}(k_\ell,z)$$

tSZ multipole-space kernel (with Gaussian beam)

Eq. 32 $$\bar{u}^j_y(\ell,z,M,c) = b^j(\ell)\cdot\frac{4\pi r_{200c}}{\ell_{200c}^2}\cdot \frac{\sigma_T}{m_e c^2}\int dx\;x^2\;P_e(x,z,M,c)\;\frac{\sin(\ell x/\ell_{200c})}{\ell x/\ell_{200c}}$$ $$b^j(\ell) = \exp\!\left[-\frac{\ell(\ell+1)\sigma_j^2}{2}\right], \quad \sigma_j = \frac{\theta^\mathrm{FWHM,j}}{\sqrt{8\ln 2}}$$

Weak lensing convergence kernel

Eqs. 34–35 $$\bar{u}^i_\kappa(\ell,z,M,c) = \frac{W^i_\kappa(z)}{\chi^2}\;u_\mathrm{dmb}(k_\ell,z,M,c)$$ $$W^i_\kappa(z) = \frac{3H_0^2\Omega_m}{2c^2}\;\frac{\chi}{a(\chi)} \int_\chi^\infty d\chi'\;n^i_\kappa(z(\chi'))\;\frac{\chi'-\chi}{\chi'}$$

Real-space correlations (Hankel transforms via mcfitjax)

Eqs. 38–39 $$\xi^i_{\gamma_t y}(\theta) = \int\frac{d\ell\,\ell}{2\pi}\;J_2(\ell\theta)\;C^i_{\kappa y}(\ell)$$ $$\xi^{ij}_\pm(\theta) = \int\frac{d\ell\,\ell}{2\pi}\;J_{0/4}(\ell\theta)\;C^{ij}_{\kappa\kappa}(\ell)$$

These Hankel integrals are computed efficiently using the mcfitjax library (JAX port of mcfit) via Mellin-transform convolutions.

Observational Systematics

Intrinsic alignment (NLA model)

Eq. 41 $$A_\mathrm{IA}(z) = -A_\mathrm{IA}(1+z)^{\eta_\mathrm{IA}}\;\frac{C_1\bar\rho_{m,0}}{D(z)}$$

Photometric redshift shift

Eq. 42 $$n^i_\kappa(z) \;\longrightarrow\; n^i_\kappa(z - \Delta^i_z)$$

Multiplicative shear bias

Eqs. 43–44 $$\xi^i_{\gamma_t y} \to (1+m^i)\,\xi^i_{\gamma_t y}, \qquad \xi^{ij}_\pm \to (1+m^i)(1+m^j)\,\xi^{ij}_\pm$$

Fisher forecast (paper)

The paper uses a Fisher matrix forecast rather than full MCMC: $$\mathbb{F}_{ab} = \frac{\partial \boldsymbol{\mu}^\top}{\partial p_a}\,\mathbf{C}^{-1}\,\frac{\partial \boldsymbol{\mu}}{\partial p_b}, \qquad \sigma^2(p_a) = (\mathbb{F}^{-1})_{aa}$$ Derivatives $\partial\boldsymbol{\mu}/\partial p_a$ are computed via jax.jacfwd (forward-mode automatic differentiation). MCMC is available in the code but was not used for the paper results.

    # 1. Sample parameters from priors
    Om0    = numpyro.sample('Om0',    dist.Uniform(prior_min, prior_max))

                                                  cov_total))

tSZ Auto-Power Spectrum — C_yy Extraction

The extract-yy branch moves the tSZ auto angular power spectrum $C_\ell^{yy}$ out of get_cov's internal calculation and into get_Cl, so that it is available as a first-class output — alongside $C_\ell^{\kappa\kappa}$, $C_\ell^{\kappa y}$, etc. — whenever get_Cl is instantiated with model_tSZ=True.

Why this matters: Previously, getting $C_\ell^{yy}$ required instantiating the full covariance class (get_cov), which is significantly slower. The extraction makes it trivial to compute just the signal spectrum for plotting, likelihood evaluation, or external comparisons without paying the cost of the trispectrum calculation.

Physics

3D tSZ auto-power spectrum

The 3D Compton-$y$ power spectrum is split into a one-halo and two-halo term in the usual halo-model way:

$$P_{yy}(k,z) \;=\; P_{yy}^{1h}(k,z) \;+\; P_{yy}^{2h}(k,z)$$ $$P_{yy}^{1h}(k,z) = \int d\ln M\;\frac{dn}{d\ln M}\;\bigl|\tilde{u}_y(k,M,z)\bigr|^2$$ $$P_{yy}^{2h}(k,z) = \left[\int d\ln M\;\frac{dn}{d\ln M}\;b(M,z)\;\tilde{u}_y(k,M,z)\right]^2 P_\mathrm{lin}(k,z) \;\equiv\; b_y^2(k,z)\;P_\mathrm{lin}(k,z)$$

where $\tilde{u}_y(k,M,z)$ is the Fourier transform of the 3D Compton-$y$ profile (pressure profile converted via $y = \tfrac{\sigma_T}{m_e c^2}\int P_e\,dl$), evaluated on the code's $(n_k, n_z)$ grid.

Beam and Limber-space kernel

Before Limber projection, the 3D power is evaluated at $k = (\ell+\tfrac{1}{2})/\chi$ and multiplied by the squared Gaussian beam $B_\ell^2$ (beam is applied twice for an auto-correlation):

$$\tilde{P}_{yy}^{\ell z}(\ell,z) \;=\; B_\ell^2\;\cdot\;P_{yy}\!\left(\frac{\ell+\tfrac{1}{2}}{\chi(z)},\,z\right), \qquad B_\ell = \exp\!\left[-\tfrac{1}{2}\ell(\ell+1)\sigma_\mathrm{beam}^2\right]$$

This is stored in Pkyy_lz_mat (shape (nell, nz)) and cached via a 2D log-interpolator.

Limber projection → $C_\ell^{yy}$

The tSZ line-of-sight window function is simply $W_y(z) = 1/(1+z)$, and the angular power spectrum is obtained by the standard Limber integral:

$$C_\ell^{yy} \;=\; \int_0^{z_\mathrm{max}} dz\;\frac{d\chi}{dz}\; \frac{W_y^2(z)}{\chi^2(z)}\;\tilde{P}_{yy}^{\ell z}(\ell,z), \qquad W_y(z) = \frac{1}{1+z}$$

This is the same integral as for every other probe combination — implemented in get_Cl_tot(jb1, jb2, probe1, probe2). By filling cached_power_spectra[3,3] with $\tilde{P}_{yy}^{\ell z}$, calling get_Cl_tot(0, 0, 3, 3) gives $C_\ell^{yy}$ through exactly the same code path as $C_\ell^{\kappa\kappa}$ = get_Cl_tot(jb1,jb2,0,0).

Before vs After

Previous implementation (inside get_cov)

$C_\ell^{yy}$ was computed privately during get_cov.__init__ via two dedicated methods that did not share code with the rest of the Limber pipeline:

  • get_Pyy_interp(jl, jz) — beam-convolved $\tilde{P}_{yy}$ at $(\ell_j, z_j)$, not stored as a class attribute
  • get_Cl_y_y_tot(jl) — dedicated scalar Limber integral, separate from get_Cl_tot
  • get_cov inherited from get_yy, which was just an alias get_yy = get_Cl inside extract_yy.py
# Old usage — required instantiating get_cov to get C_yy
from godmax.get_covs import get_cov   # (inherited from get_yy alias)

cov = get_cov(sim_params_dict, halo_params_dict, analysis_dict, other_params_dict)
Cl_yy = np.array(cov.Cl_y_y_tot_mat)   # only available here
Problems with the old approach:
  • Getting $C_\ell^{yy}$ required the full (slow) covariance computation
  • extract_yy.py was a superfluous wrapper file with just one alias line
  • Two separate Limber integral implementations for the same physics — maintenance burden
  • Pkyy_lz_mat was never stored as an attribute — no way to inspect the beam-convolved 3D spectrum

New implementation (extract-yy branch)

$C_\ell^{yy}$ is now computed inside get_Cl.__init__ under the if self.model_tSZ: block, using the same get_Cl_tot path as every other probe: 

# Inside get_Cl.__init__  (src/godmax/get_Cls.py)
if self.model_tSZ:
    # 1. Compute P_yy(k,z) = 1h + 2h
    vmapped_func_yy  = get_vmapped_func_warg(self.get_P_1h, 2, 4)
    Pyy_1h_kz_mat    = vmapped_func_yy(jnp.arange(self.nk), jnp.arange(self.nz), 3, 3).T
    Pyy_2h_kz_mat    = self.by_kz_mat ** 2 * self.plin_kz_mat
    self.Pyy_tot_kz_mat = Pyy_1h_kz_mat + Pyy_2h_kz_mat   # (nk, nz)

    # 2. Evaluate at Limber k=(ℓ+0.5)/χ, apply beam², build 2D interpolator
    self.Pkyy_lz_mat = get_vmapped_func(self.get_Pkyy_lz, 2)(
        jnp.arange(self.nell), jnp.arange(self.nz)).T       # (nell, nz)
    self.logPkyylz_2d_interp = interpax.Interpolator2D(
        jnp.log(self.ell_array), self.z_array,
        jnp.log(self.Pkyy_lz_mat), extrap=True)

    # 3. Fill unified cache — same structure as κκ, κy, gg, …
    self.cached_power_spectra = self.cached_power_spectra.at[3, 3].set(
        vmap(lambda l: jnp.exp(self.logPkyylz_2d_interp(l, self.z_array_for_Cls)))(log_ell))

    # 4. Limber integral via the same get_Cl_tot path
    self.Cl_y_y_tot_mat = self.get_Cl_tot(0, 0, 3, 3)      # (nell,)

The helper method get_Pkyy_lz(jl, jz) (JIT-compiled) interpolates $P_{yy}$ onto the Limber wavenumber and applies the beam: 

@partial(jit, static_argnums=(0,))
def get_Pkyy_lz(self, jl, jz):
    ell      = self.ell_array[jl]
    chi_z    = self.chi_array[jz]
    Bl       = jnp.exp(-0.5 * ell * (ell + 1) * self.sig_beam**2)
    k_ell    = (ell + 0.5) / jnp.clip(chi_z, 1.0)
    Pkz_ell  = jnp.exp(jnp.interp(
        jnp.log(k_ell),
        jnp.log(self.kPk_array),
        jnp.log(self.Pyy_tot_kz_mat[:, jz])))
    return Bl**2 * Pkz_ell                   # beam² for auto-correlation
File-level changes:
  • src/godmax/extract_yy.py — deleted (was just get_yy = get_Cl)
  • src/godmax/__init__.py — emptied (removed the get_yy import)
  • src/godmax/get_Cls.py — added P_yy 3D calc, Pkyy_lz_mat, cached_power_spectra[3,3], get_Pkyy_lz method, and Cl_y_y_tot_mat
  • src/godmax/get_covs.py — changed class get_cov(get_yy) → class get_cov(get_Cl); simplified else branch to self.__dict__.update(Cl_obj.__dict__)

How $C_\ell^{yy}$ feeds into the covariance

Inside get_cov, the yy signal is used in two places:

  1. Gaussian covariance (Knox formula)
    Cl_result_dict['yy']['bin_0_0']['tot_plus_noise_ellsurvey'] $= C_\ell^{yy} + N_\ell^{yy}$ is used to build the diagonal Gaussian covariance: $$\mathrm{Cov}^G_{\ell\ell'} = \frac{2\delta_{\ell\ell'}}{(2\ell+1)\,f_\mathrm{sky}\,\Delta\ell} \left(C_\ell^{yy} + N_\ell^{yy}\right)^2$$
  2. Non-Gaussian trispectrum (1-halo)
    ul_dict['y_0'] = $\tilde{u}_y(\ell)\,W_y/\chi^2$ is the Limber-projected tSZ profile kernel, used to form the connected 4-point term: $$T^{1h}(\ell_1,\ell_2) = \int dz\;\frac{d\chi}{dz}\,\chi^{-6} \int d\ln M\;\frac{dn}{d\ln M}\;\bigl|\tilde{u}_y(\ell_1)\bigr|^2\bigl|\tilde{u}_y(\ell_2)\bigr|^2$$

Usage

With the new implementation, $C_\ell^{yy}$ is obtained directly from get_Cl:

import numpy as np
from godmax.get_Cls  import get_Cl
from godmax.get_covs import get_cov

# ── 1. Instantiate get_Cl (computes C_yy automatically when model_tSZ=True) ──
cl = get_Cl(sim_params_dict, halo_params_dict, analysis_dict, other_params_dict)

# ── 2. Access the outputs ──
Cl_yy      = np.array(cl.Cl_y_y_tot_mat)    # (nell,)   angular C_yy
Pyy_3d     = np.array(cl.Pyy_tot_kz_mat)    # (nk, nz)  3D P_yy = 1h + 2h
Pkyy_limber= np.array(cl.Pkyy_lz_mat)       # (nell, nz) beam²·P_yy at Limber k

# Also available (unchanged):
Cl_ky      = np.array(cl.Cl_kappa_y_tot_mat)     # (nbins, nell)
Cl_kk      = np.array(cl.Cl_kappa_kappa_tot_mat) # (nbins, nbins, nell)

# ── 3. Reuse get_Cl object for covariance (avoids recomputation) ──
cov = get_cov(sim_params_dict, halo_params_dict, analysis_dict, other_params_dict,
              Cl_obj=cl)

# Access covariance matrices
covG  = cov.covG_dict ['yy_yy']['bin_0_0_0_0']   # (nell, nell) Gaussian
covNG = cov.covNG_dict['yy_yy']['bin_0_0_0_0']   # (nell, nell) non-Gaussian
covtot= cov.covtot_dict['yy_yy']['bin_0_0_0_0']  # (nell, nell) total
Things to note when using C_yy:
  • Requires model_tSZ: true in your analysis_dict (or param YAML), otherwise the if self.model_tSZ: block is skipped and Cl_y_y_tot_mat will not exist
  • The beam ($\sigma_\mathrm{beam}$) is set via sim_params_dict; for no beam, set sig_beam = 0
  • Cl_y_y_tot_mat has shape (nell,) — it is a scalar probe (no tomographic bins), unlike $C_\ell^{\kappa\kappa}$ which has shape (nbins, nbins, nell)
  • The yy noise spectrum must be provided via analysis_dict['yy_noise_ell_fname'] pointing to a two-column file (ell, $N_\ell^{yy}$); this is only used in the covariance, not in the signal
  • Passing Cl_obj=cl to get_cov reuses the pre-computed get_Cl and skips recomputation — always do this to save time
