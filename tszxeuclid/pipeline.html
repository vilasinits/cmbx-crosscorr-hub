The thermal Sunyaev–Zel'dovich effect

CMB photons travelling through the hot plasma of galaxy clusters and filaments experience inverse-Compton scattering off electrons with temperatures $T_e \sim 10^7$–$10^8\,\text{K}$. The amplitude of the distortion is the Compton-$y$ parameter: the line-of-sight integral of electron pressure:

$$y(\hat{n}) = \frac{\sigma_T}{m_e c^2}\int P_e(\hat{n},\ell)\,d\ell, \qquad P_e = n_e\,k_B T_e$$

$\sigma_T = 6.65\times10^{-29}\,\text{m}^2$  |  $m_ec^2 = 511\,\text{keV}$  |  Spectral signature: decrement at $\nu\lesssim217\,\text{GHz}$, increment above — redshift-independent in amplitude.

Why cross-correlate with Euclid?

tSZ probes hot gas pressure along the line of sight, tracing massive halos. Euclid's galaxy clustering traces the underlying matter distribution via galaxy bias, while weak lensing probes the total projected matter density. Together they break parameter degeneracies:

  • tSZ $\propto \sigma_8^7\,\Omega_m^2$ — extremely sensitive to structure growth
  • Galaxy clustering $\propto \sigma_8\,b_g$ — constrains bias
  • Weak lensing $\propto \sigma_8\,\Omega_m$ — constrains matter density
  • Cross-correlations cancel uncorrelated noise (CIB, photo-$z$ scatter)

GODMAX — Gas thermODynamics and Matter distribution using jAX. A GPU-accelerated halo model framework for Bayesian inference on angular cross-power spectra. arXiv:2401.18072. Current branch: refactor-map.
JAX · GPU NumPyro NUTS pip install refactor-map branch

What it does

Baryonic feedback (AGN jets, supernova winds) pushes gas out of halos, suppressing the matter power spectrum by 10–20% at the scales probed by weak lensing. GODMAX models this self-consistently by solving the hydrostatic equilibrium (HSE) ODE — rather than applying a post-hoc fitting function — and derives the electron pressure $P_e(r)$ directly from the coupled density and temperature profiles.

HSE = hydrostatic equilibrium. The full pressure profile is derived from the gas density profile, not assumed parametrically.

Key equations

Total power spectrum = 1-halo (cluster interior) + 2-halo (between clusters):

$$P^{AB}_{\rm tot}(k,z) = P^{AB}_{\rm 1h}(k,z) + P^{AB}_{\rm 2h}(k,z)$$

The 1-halo term integrates over the halo mass function $dn/dM$ and concentration distribution:

$$P^{AB}_{\rm 1h}(k,z) = \int dM\frac{dn}{dM}\int dc\,p(c|M)\;\tilde{u}_A\,\tilde{u}_B$$

The tSZ profile in harmonic space (with instrumental beam $b_j$):

$$\tilde{u}_y(\ell,z,M) = b_j(\ell)\;\frac{\sigma_T}{m_e c^2}\; \frac{4\pi r_{200c}}{\ell_{200c}^2} \int x^2 P_e(x)\;\frac{\sin(\ell x/\ell_{200c})}{\ell x/\ell_{200c}}\,dx$$

Inference with NUTS

GODMAX is differentiable end-to-end via JAX. This enables Hamiltonian Monte Carlo (HMC) — which uses gradient information to propose efficient moves in parameter space — and in particular the NUTS (No-U-Turn Sampler) variant that adapts trajectory length automatically.

Why NUTS for this problem?

  • Parameters are correlated: $\sigma_8$, $\Omega_m$, $b_{\rm HSE}$, $P_0$ — HMC handles this natively with a dense mass matrix
  • Gradient computed automatically by JAX autodiff (no finite differences)
  • JIT compilation makes each likelihood call ~1 ms on GPU
  • Multi-chain via jax.pmap: one chain per GPU, zero communication overhead

Typical NUTS settings

kernel = NUTS(
  model,
  dense_mass = True,   # full mass matrix
  max_tree_depth = 10
)
mcmc = MCMC(
  kernel,
  num_warmup  = 500,
  num_samples = 1000,
  num_chains  = 4      # one per GPU via pmap
)

Euclid inference stack

cloelib & cloelike

The protocol-based Python stack that converts cosmological parameters into Euclid primary-probe log-likelihoods. Boltzmann solver is swappable at runtime; JAX throughout for automatic differentiation.

cloelib

Core observable library. Defines Python Protocols for Background, Perturbations, and Tracer — CAMB, CLASS, JAX, or a custom emulator are all drop-in replacements. Computes lensing/clustering window functions and angular power spectra $C_\ell$ via the Limber approximation using JAX vmap + Simpson integration.

Python ≥ 3.10 JAX · autodiff hatchling · pre-commit · ruff CAMB · CLASS · JAX · CometEFT backends

cloelike

Likelihood layer on top of cloelib. Provides mixin-based classes for WL, GCph, GGL (3×2pt), spectroscopic power multipoles $P_\ell(k)$, and BAO. All share the same loglike(params → float) interface and accept any Background/Perturbations implementation satisfying the Protocols.

depends on cloelib 3×2pt · BAO · GCspectro χ² = ΔᵀC⁻¹Δ

$$C_\ell^{ij} = \frac{c}{H_0}\int_0^{z_{\max}} dz\; \frac{W_i(z)\,W_j(z)}{H(z)\,\chi^2(z)}\; P_m\!\left(\frac{\ell+\tfrac12}{\chi(z)},\,z\right)$$

Relevance for GODMAX tSZ likelihood in dr1_cmbx

The cloelib/cloelike pattern is the blueprint for implementing a GODMAX-based tSZ likelihood in dr1_cmbx. The table below maps each cloelib concept to its GODMAX counterpart and identifies what actually needs to be built.

What the package does

GODMAX computes all cross-power spectra between galaxy clustering, CMB weak lensing, and the tSZ Compton-y field using a self-consistent baryonic halo model. Unlike approaches that add baryonic corrections post-hoc, GODMAX solves the hydrostatic equilibrium ODE for each halo to derive the pressure profile directly — enabling self-consistent parameter inference on both cosmology and baryonic feedback simultaneously.

How the class hierarchy works

Each class in the inheritance chain adds one layer of computation and stores results as instance attributes. You only instantiate the highest level you need. 

base_class        # reads 4 YAML dicts; sets up cosmology, grids, P_lin(k,z)
  └─► Profiles      # computes all halo profiles as (nz × nM × nr) matrices via vmap
        └─► get_Pkz # assembles 3D P(k,z): 1h = ∫u_A u_B dn/dlnM dM, 2h = b_A b_B P_lin
              └─► get_Cl  # Limber: C_ℓ = ∫ W_A W_B / χ² × P(ℓ/χ, z) dz
                    ├─► get_cov  # Gaussian (Knox) + 1h non-Gaussian (trispectrum) covariance
                    └─► get_xi   # real-space ξ(θ) via Hankel transform

The optional *_obj=None argument on each constructor lets you pass a pre-computed lower-level object to avoid recomputation when sweeping parameters.

Key design decisions that enable JAX autodiff

  • Symbolic HMF (hmf_symbolic.py): the Tinker HMF requires $\sigma(M,z)$, which normally needs CAMB. A symbolic regression fit to $\sigma(R,z)$ makes the entire HMF computation a closed-form JAX expression — the autodiff graph stays intact for NUTS gradient computation.
  • Symbolic power spectrum (matter_pk_symbolic.py): same idea for $P_{\rm lin}(k,z)$, $P_{\rm halofit}(k,z)$, and $D(z)$ — no external CAMB call inside the inference loop.
  • vmap pattern: all profile integrals are written as scalar functions @jit def get_X(self, jz, jM, jr), then vmapped over all grid indices simultaneously via get_vmapped_func(self.get_X, 3)(arange(nz), arange(nM), arange(nr)). This produces GPU-accelerated lookup matrices.
  • mcfitjax: a JAX port of the mcfit Mellin–Hankel transform library. Used to Fourier-transform radial profiles $u(r) \to \tilde{u}(k)$ and to compute real-space correlation functions $\xi(\theta)$ — both differentiably.

How the yy auto-spectrum is computed in get_cov.__init__(): 

# 1. Build the 3D tSZ auto-power P_yy(k,z) = P_yy_1h + b_y^2 * P_lin
Pyy_1h_kz_mat  = vmap(get_P_1h, probe1=3, probe2=3)   # shape (nk, nz)
Pyy_2h_kz_mat  = by_kz_mat**2  *  plin_kz_mat
self.Pyy_tot_kz_mat = Pyy_1h_kz_mat + Pyy_2h_kz_mat

# 2. Project to (ℓ, z) with beam suppression B(ℓ)²
def get_Pkyy_lz(jl, jz):
    ell   = ell_array[jl]
    Bl    = exp(-ell*(ell+1) * sig_beam**2 / 2.)     # Gaussian beam
    k_ell = (ell + 0.5) / chi_array[jz]              # Limber k
    Pkz   = interp(log(k_ell), log(kPk_array), log(Pyy_tot[:,jz]))
    return Bl**2  *  exp(Pkz)

# 3. Angular Cℓ^yy via Limber integral
def get_Cl_y_y_tot(jl):
    Wy = 1 / (1 + z_array)                           # tSZ window function
    integrand = Wy**2 / chi**2  *  P_yy_limber  *  chi**2  *  dchi_dz
    return trapezoid(integrand, z_array)

Noise handling: if yy_total_ell_fname is provided, the total (signal + noise) $C_\ell^{yy}$ is loaded directly from file and interpolated onto the survey ell array. Otherwise a noise file can be added to the theory prediction. The non-Gaussian covariance for yy uses the 1-halo trispectrum $T^{yyyy}$ via get_T_ABCD_NG with all four legs set to probe=3 (tSZ).

Key changes — before & after

ConcernBefore — inline in notebook cellsAfter — imported from package
Parameter loading Two separate yaml.safe_load() calls merged with {**default, **experiment} — shallow merge silently drops any nested keys that exist in default but not in experiment merge_params(PARAM_DIR/"params_default.yaml", PARAM_DIR/"Pge/params.yaml") — deep recursive merge via deepmerge.always_merger; nested keys are preserved and overridden correctly
File paths Absolute paths hardcoded: abs_path_params = "/home/ldumilly/tSZxEuclid/…" — breaks immediately on any other machine PARAM_DIR = Path("../../../params_godmax").resolve() — path is relative to notebook location, resolved at runtime; works on any machine or as GODMAX_OUTPUT_DIR environment variable on HPC
ℓ-array setup 5-line cell: l_edges = np.exp(np.arange(np.log(lmin), np.log(lmax), dl_log)), then manually assigned into both analysis and halo dicts setup_ell_arrays(analysis, halo, lmin=80, lmax=500, dl_log=0.53) — single call; handles log-spacing, bin edges, and both dict assignments
Galaxy $n(z)$ Inline def to_comoving_nz(z, nz, cosmo): … defined in a cell, then np.loadtxt(…) + 3 manual dict assignments per bin setup_comoving_nz(DATA_DIR/"euclid_nz.txt", cosmo, analysis) — loads, converts to comoving units, and stores in analysis dict; setup_lens_bins / setup_source_bins for tomographic splits
Bin combinations 20-line for probe in probes_analysis: loop with if probe in ['gy', 'ky']: / else: branching to decide which array axes to iterate over bin_comb_all = build_bin_combinations(probes, Cl_theory) — returns an ordered list of (probe, bin1, bin2) tuples; probe-agnostic
Flat data vector 15-line loop using JAX .at[i*nell:(i+1)*nell].set(…) with 5 if probe == branches selecting the right array slice per probe data_vec = build_data_vector(get_cl, bin_comb_all, nell) — iterates over bin_comb_all, automatically handles all probe types; output is a flat JAX array of shape (ncomb × nell,)
Block covariance 25-line def full_cov_matrix(…): defined inline inside a notebook cell, re-implemented for each new analysis cov_total = full_cov_matrix(cov, bin_comb_all, nell) from tSZxEuclid.godmax.sampling.model
MCMC execution Inline def do_mcmc(): … closure capturing outer-scope variables, then pmap(do_mcmc)(rng_keys), then manual for key in trace: concatenate along axis 0 loop trace = run_pmap_mcmc(observed_model, num_warmup=500, num_samples=1000, num_chains=4, max_tree_depth=10, seed=42) — handles pmap, chain concatenation, and RNG key splitting internally
Output saving Hardcoded path /home/ldumilly/GODMAX/runs/…, manual pickle.dump save_chain(trace, OUTPUT_DIR, "run_pge_v1.pkl") via io.py; also attaches run metadata (add_run_settings) and supports load_chain for downstream diagnostics

What this enables

  • Notebooks are now reproducible on any machine: set PARAM_DIR and DATA_DIR at the top; all function calls work unchanged
  • HPC scripts (scripts/python/godmax/) can import the same functions without duplicating logic
  • Unit tests (tests/) verify each module independently — previously impossible with inline code
  • Post-processing (diagnostics, plotting) is now a single function call on a loaded chain file
